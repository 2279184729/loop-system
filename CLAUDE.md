# CLAUDE.md - 项目说明

## 项目概述

**Claude Code 父子多层嵌套自适应Loop系统** v2.5，全栈高阶多Agent方案。

核心解决的问题：**Claude Code 原生模式在处理复杂任务时，所有子任务的代码、执行结果、审查信息全挤在一个上下文窗口里，导致上下文膨胀、细节丢失、甚至中断。
** 本系统通过独立 `claude` CLI 进程实现真正的上下文隔离，每个子 Agent 拥有全新的上下文窗口。

所有任务统一走四阶段复杂任务流程：**规划 → 并行调度 → 合并 → 校验**。

## 使用方式

```bash
# 多Agent上下文隔离并行
python orchestrator.py "从零搭建全栈用户管理系统"

# 产品迭代循环
python iterate.py --max-iterations 5 "为项目添加用户认证和日志系统"

# 项目扫描
python project_scanner.py . --summary
```

## 核心架构

```
loop-system/
├── orchestrator.py       # 父调度Agent（规划→调度→合并→校验）
├── config.py             # 全局配置 + 系统提示词
├── utils.py              # claude CLI 调用封装 + 工具函数
└── workspaces/           # 子Agent独立工作区
```

## 四阶段流程

```
阶段一：全局方案规划 → claude CLI 生成任务拆分方案（max 5轮迭代）
阶段二：分布式调度   → ThreadPoolExecutor 并行启动 claude CLI 子进程（依赖排序分批）
阶段三：结果回流汇总 → 从子工作区复制文件到项目目录 + diff冲突检测 + 语法校验
阶段四：全局自检修复 → claude CLI 代码审查 + 自动修复（max 5轮迭代）
```

## 上下文隔离原理

```
Claude Code 原生模式:              本系统 CLI 模式:
┌──────────────────────┐           ┌──────────────────────┐
│ 单个上下文窗口         │           │ 父进程（轻量上下文）    │
│ ├── 规划方案           │           │ ├── 计划摘要           │
│ ├── 子任务1 全部代码    │           │ └── 各子任务结果摘要    │
│ ├── 子任务2 全部代码    │  ← 膨胀   └──────────────────────┘
│ ├── 子任务3 全部代码    │           ┌──────────────────────┐
│ ├── 合并冲突           │           │ claude CLI 子进程1     │
│ └── 审查结果           │           │ → 全新窗口 → 模块1代码  │
│ → 超出上限 → 压缩/中断  │           └──────────────────────┘
└──────────────────────┘           ┌──────────────────────┐
                                   │ claude CLI 子进程2     │
                                   │ → 全新窗口 → 模块2代码  │
                                   └──────────────────────┘
```
