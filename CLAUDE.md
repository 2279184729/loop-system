# CLAUDE.md - 项目说明

## 项目概述

**Claude Code 父子多层嵌套自适应Loop系统** v2.4，全栈高阶多Agent方案。

核心解决的问题：**Claude Code 原生模式在处理复杂任务时，所有子任务的代码、执行结果、审查信息全挤在一个上下文窗口里，导致上下文膨胀、细节丢失、甚至中断。** 本系统通过独立进程 + 独立 API 调用实现真正的上下文隔离，每个子 Agent 拥有全新的上下文窗口。

## 两种使用模式

### 模式A：Python 脚本模式（推荐用于复杂任务）

每个子 Agent 是独立进程，拥有独立的 Claude API 上下文窗口。父进程只拿结果摘要，不装子任务细节。适合多模块、跨文件、大规模项目。

```bash
# 简单任务 — 单 Agent 直出
python orchestrator.py "修改 backend/main.py 的端口从8000改为9000"

# 复杂任务 — 多 Agent 上下文隔离并行
python orchestrator.py "从零搭建全栈用户管理系统"

# 产品迭代循环
python iterate.py --max-iterations 5 "为项目添加用户认证和日志系统"

# 项目扫描
python project_scanner.py . --summary
```

### 模式B：Claude Code 原生模式（适合简单/中等任务）

直接在对话中说"用loop方法论"或"按loop流程"，Claude Code 充当 Orchestrator。优势是零配置、交互式，但所有内容共享单个上下文窗口，复杂任务容易触发压缩或中断。

## 模式选择指南

| 场景 | 推荐模式 | 原因 |
|---|---|---|
| 单文件修改、小修复 | 模式B | 简单直接，无需启动子进程 |
| 2-3个文件的改动 | 模式B | 上下文够用 |
| 全栈项目、多模块开发 | 模式A | 上下文隔离，不会膨胀 |
| 需要断点续传 | 模式A | 独立进程可中断恢复 |
| 批量自动化、CI/CD | 模式A | 可脚本化，无交互依赖 |

## 核心架构

```
loop-system/
├── orchestrator.py       # 父调度Agent（复杂度判定→规划→调度→合并→校验）
├── child_agent.py        # 子执行Agent（独立进程，独立API上下文）
├── project_scanner.py    # 项目上下文扫描
├── iterate.py            # 产品迭代循环引擎
├── config.py             # 全局配置 + 工具定义 + 系统提示词
├── utils.py              # Claude API客户端 + Agent Loop引擎 + 工具执行
└── workspaces/           # 子Agent独立工作区
```

## 上下文隔离原理

```
Claude Code 原生模式:              本系统 Python 模式:
┌──────────────────────┐           ┌──────────────────────┐
│ 单个上下文窗口         │           │ 父进程（轻量上下文）    │
│ ├── 规划方案           │           │ ├── 计划摘要           │
│ ├── 子任务1 全部代码    │           │ └── 各子任务结果摘要    │
│ ├── 子任务2 全部代码    │  ← 膨胀   └──────────────────────┘
│ ├── 子任务3 全部代码    │           ┌──────────────────────┐
│ ├── 合并冲突           │           │ 子进程1（独立上下文）   │
│ └── 审查结果           │           │ → 全新窗口 → 模块1代码  │
│ → 超出上限 → 压缩/中断  │           └──────────────────────┘
└──────────────────────┘           ┌──────────────────────┐
                                   │ 子进程2（独立上下文）   │
                                   │ → 全新窗口 → 模块2代码  │
                                   └──────────────────────┘
```
